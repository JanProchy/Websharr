Help

How Websharr fits into your media stack, and how to wire it into Sonarr / Radarr.

How the flow works

You request something in Jellyseerr → Sonarr/Radarr search through Websharr's Newznab indexer (directly or via Prowlarr) → on grab, Websharr's SABnzbd download client pulls the file from your Webshare premium account → Sonarr/Radarr import and rename it into the library → Jellyfin plays it. For Czech content Websharr looks the original title up in TMDB so it can match the *arr request to the Czech-named Webshare file. Websharr is the same container playing every role.

1 · Add the indexer  Newznab

Sonarr/Radarr → Settings → Indexers → + → Newznab (Custom). Add it as Newznab, not Torznab — Torznab is treated as torrent and its grabs would go to a torrent client instead of Websharr.

Czech titles  TMDB

Sonarr/Radarr search by the title they know — usually English — but Webshare files are named in Czech, so those searches find nothing. Add a TMDB Read Access Token in Settings and Websharr looks each request up in TMDB and searches Webshare under its original (Czech) title and its Czech alternative titles (the dub name of a foreign show, e.g. Kačeří příběhy for DuckTales), while keeping the canonical name for the release — so it grabs Bez vědomí but reports The Sleepers.

  • ID lookups (TVDB/IMDb/TMDB) are tried first, then a name match. Behind Prowlarr only the name match applies — Prowlarr forwards IDs as plain text.
  • Search aliases (Settings) are a manual override — map a title to the exact Webshare name when TMDB has no Czech title or picks the wrong match.
  • The token is optional: without it, aliases still work and everything else runs normally.

Good to know

  • Host names like websharr work when everything is on the same Docker network. Otherwise use the LAN IP and port.
  • Websharr and Sonarr/Radarr must see the same download folder so imports find the files — mount the same path, or set a Remote Path Mapping in *arr.
  • Webshare searches file names only. Websharr normalizes TV releases to Series SxxEyy so most imports work, but the occasional oddly-named CZ file still needs a manual import in Sonarr.
  • Watch the Log tab to see the *arr requests arrive in real time.

Settings

Webshare.cz account

The premium account used for searching and downloading. Webshare.cz has no API key — just this username/e-mail and password.

Websharr API key

Generated by Websharr — paste it into the API Key field of the Torznab indexer and the SABnzbd download client in Sonarr/Radarr. This is not a Webshare.cz key.

Downloads

How many files download from Webshare at the same time. Extra grabs wait in the queue. You can also change this right on the Queue page.

Notifications

Get alerted on failed downloads, when Webshare is unreachable, and when your VIP is about to expire. One Apprise URL per line (Discord, Telegram, ntfy, Slack, Gotify, …).

Czech titles (TMDB)

Sonarr/Radarr search by the title they know (usually English), but many Webshare files use the Czech name. With a TMDB API Read Access Token, Websharr looks up the Czech title automatically (by id when Sonarr/Radarr send it, otherwise by name) and searches it too.

Search aliases

Manual override/supplement to the TMDB lookup. Map *arr title → Webshare (Czech) title and Websharr also searches the Czech name. Use this when TMDB has no Czech title or picks the wrong match. Matching is loose — the series/movie name is enough.

Storage paths

Finished downloads land in completed as <category>/<release>/<file>. These are paths inside the container — check the /downloads volume in docker-compose.yml to see where they are on the host.

Websharr account

The account you sign in to this interface with.
